## 📖 痛点洞察与产品定位

针对缺乏稳定雇佣关系的零工从业者（如 Uber/DoorDash 司机），其核心财务痛点并非缺乏记账工具，而是跨平台收入波动大与刚性支出之间的冲突。面对多平台接单的决策犹豫，他们极度缺乏“财务安全感” 。

本项目（Visa GigOS）定位为一款基于 Agentic AI 架构的财务智能体平台。旨在将用户的核心财务动作从“记录过去的被动记账”，升级为“规划未来的主动现金流管理” 。


## 🧠 核心产品机制 (Core Mechanisms)

本 MVP 摒弃了单一巨型模型，构建了基于“感知 - 规划 - 行动”的多智能体协同架构（Multi-Agent System） ：

1. **防御型 Agent（风险评估）：** 实时感知账户数据波动，精准计算资金缺口并发出预警信号 。搭配“全景式智能账单管理”，在资金充足时自动支付固定账单，降低滞纳金风险。
2. **进攻型 Agent（目标规划）：** 接收预警后激活，结合天气（如降雨）、本地事件等外部环境因子，主动推算并生成“高收益排班填补方案”。
3. **执行型 Agent（自动执行）：** 严格遵循 Human-in-the-loop（人在回路）机制 。系统提供可视化的行动建议，用户保留最终排班与支付的决策权。


## 📈 商业逻辑与北极星指标

* **北极星指标 (North Star Metric)：** 设定为月均日均存款余额 (ADB) 。该指标平滑了零工收入的波动，既衡量了用户的财务健康度，又体现了资金沉淀的商业价值 。
* **Visa 生态赋能：** 通过高频自动支付增加交易手续费收入；同时帮助零工群体建立“替代性数据”信用档案，拓展普惠金融与下沉市场获客。


## 🛠️ 构建方式与技术边界 (Disclaimer)

**本项目严格界定为一次采用 Vibe Coding（意图编程）理念的前端概念验证（Frontend PoC）。**

* **AI 驱动敏捷开发：** 在 0 研发资源介入下，通过调优 Prompt 引导大语言模型生成全部前端代码，并部署于 GitHub Pages 完成极速验证。
* **数据模拟与合规：** 考虑到企业级数据孤岛与金融合规（如 PCI-DSS）壁垒，本 Demo 中涉及的所有跨平台收入流与 Visa 账户流水均为**前端硬编码（Mock Data）模拟展示**。
* **验证目标：** 本页面的核心目的仅在于验证“主动式 AI 财务干预”与 Human-in-the-loop 交互逻辑的用户接受度（Fake Door Test），不涉及任何真实用户隐私处理。
